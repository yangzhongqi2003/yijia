
既然你的 cert-manager 已经正常运行，那么推荐直接使用 方案一（Helm 管理 Ingress + Let's Encrypt 自动签发证书）。

下面是一份可以直接用于 kube-prometheus-stack 76.5.1 的完整 values-ingress.yaml 示例。

1. 先确认 ClusterIssuer

先检查：

kubectl get clusterissuer

如果你看到类似：

NAME               READY
letsencrypt-prod   True

则下面的 YAML 可以直接使用。

如果名称不是 letsencrypt-prod，把下面 YAML 里的名字改掉。

2. 编写yaml
values-ingress.yaml

grafana:
  ingress:
    enabled: true

    ingressClassName: nginx

    annotations:
      cert-manager.io/cluster-issuer: prod-issuer

    hosts:
      - grafana-br.targeted.work

    path: /

    tls:
      - secretName: grafana-tls
        hosts:
          - grafana-br.targeted.work

prometheus:
  ingress:
    enabled: true

    ingressClassName: nginx

    annotations:
      cert-manager.io/cluster-issuer: prod-issuer

      nginx.ingress.kubernetes.io/auth-type: basic
      nginx.ingress.kubernetes.io/auth-secret: prometheus-basic-auth
      nginx.ingress.kubernetes.io/auth-realm: "Prometheus Login"

    hosts:
      - prometheus-br.targeted.work

    paths:
      - /

    tls:
      - secretName: prometheus-tls
        hosts:
          - prometheus-br.targeted.work

alertmanager:
  ingress:
    enabled: true

    ingressClassName: nginx

    annotations:
      cert-manager.io/cluster-issuer: prod-issuer

      nginx.ingress.kubernetes.io/auth-type: basic
      nginx.ingress.kubernetes.io/auth-secret: alertmanager-basic-auth
      nginx.ingress.kubernetes.io/auth-realm: "Alertmanager Login"

    hosts:
      - alertmanager-br.targeted.work

    paths:
      - /

    tls:
      - secretName: alertmanager-tls
        hosts:
          - alertmanager-br.targeted.work



3. 创建 Basic Auth

Prometheus 和 Alertmanager 不建议裸露公网。

安装工具：

Amazon Linux：

sudo yum install httpd-tools -y

Ubuntu：

sudo apt install apache2-utils -y

生成密码：

htpasswd -nb admin '1234561qaz'

例如返回：

admin:$apr1$xxxx$xxxxxxxxxxxxxxxx

创建 Prometheus Secret

kubectl create secret generic prometheus-basic-auth \
  --from-literal=auth='admin:$apr1$6MqfTtcY$KD2HouAp1RZQogEOtDS9y.' \
  -n monitoring

创建 Alertmanager Secret

kubectl create secret generic alertmanager-basic-auth \
  --from-literal=auth='admin:$apr1$6MqfTtcY$KD2HouAp1RZQogEOtDS9y.' \
  -n monitoring
4. Helm 安装

kubectl create ns monitoring

curl https://raw.githubusercontent.com/helm/helm/main/scripts/get-helm-3 | bash

helm repo add prometheus-community \
https://prometheus-community.github.io/helm-charts

helm repo update

假设你已经有：

override-images.yaml
values-prod.yaml
values-ingress.yaml

执行：





helm install monitoring \
prometheus-community/kube-prometheus-stack \
-n monitoring \
--version 76.5.1 \
-f override-images.yaml \
-f values-prod.yaml \
-f values-ingress.yaml \
-f patch-values.yaml


helm upgrade monitoring \
prometheus-community/kube-prometheus-stack \
-n monitoring \
--version 76.5.1 \
-f override-images.yaml \
-f values-prod.yaml \
-f values-ingress.yaml \
-f patch-values.yaml

kubectl rollout restart deploy \
monitoring-kube-prometheus-operator \
-n monitoring

5. DNS 配置

你的公网 NLB：

k8s-ingressn-ingressn-ebdb938bd2-e833f1dbded2fa72.elb.sa-east-1.amazonaws.com

添加 DNS：

类型	主机名	目标
CNAME	grafana.example.com	k8s-ingressn-ingressn-ebdb938bd2-e833f1dbded2fa72.elb.sa-east-1.amazonaws.com
CNAME	prometheus.example.com	k8s-ingressn-ingressn-ebdb938bd2-e833f1dbded2fa72.elb.sa-east-1.amazonaws.com
CNAME	alertmanager.example.com	k8s-ingressn-ingressn-ebdb938bd2-e833f1dbded2fa72.elb.sa-east-1.amazonaws.com
6. 验证

查看 Ingress：

kubectl get ingress -n monitoring

应该出现：

monitoring-grafana
monitoring-kube-prometheus-prometheus
monitoring-kube-prometheus-alertmanager

查看证书：

kubectl get certificate -n monitoring

正常会显示：

grafana-tls
prometheus-tls
alertmanager-tls

READY：

True
